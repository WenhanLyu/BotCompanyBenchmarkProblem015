## Root Cause Analysis

### Problem: In-Memory Cache Accumulates Unbounded Entries

With NUM_BUCKETS = 20 and no cache clearing:
1. All 100K operations distribute across only 20 buckets
2. Cache stores ALL active (index, value) pairs for each accessed bucket
3. Cache grows to ~50K entries per heavily-used bucket
4. Memory formula: ~50K × (string overhead + int + pair overhead) ≈ 40+ MiB

### Why Tests Fail

**Insert-Heavy Test:**
- 70% inserts create ~70K entries
- Distributed unevenly across 20 buckets
- Cache accumulates multiple heavy buckets in memory
- Result: 11.76 MiB (1.96x over limit)

**Collision Test:**
- Adversarial: forces many operations into few buckets
- Cache holds ~50K entries for single bucket
- Result: 41.31 MiB memory, 56.95s time
